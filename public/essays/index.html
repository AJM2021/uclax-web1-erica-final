01. Explain the difference between HTML, CSS and JS.

HTML is a programming language that provides the basic structure of websites, which can be enhanced and modified by other technologies like CSS and Javascript. HTML is at the heart of every webpage, even complex sites. Understanding HTML is a key skipp for web professionals and the beginning point for people learning to create content for the web. HTML stands for HyperText Markup Language, referring to the tags it uses to identify different types of content and the purposes they serve on webpages (“Markup language”).

CSS is a technology used to control overall presentation, formatting, and layout. CSS is an acronym for “Cascading Style Sheets.” It dictates how the HTML elements of a website should appear on a page. While HTML provides the raw language needed to structure content on a page, CSS helps to style the content to appear the way the designer wishes it to. The languages are separate to ensure websites are built correctly (HTML) before they are formatted (CSS).

JavaScript is used to control the behavior of different items on the page. It is a more complicated and more recent language than HTML or CSS. It is used on almost every site for more powerful and complete functionality. JavaScript can be used to modify page content and make it behave in response to a user’s action. Common uses include confirmation boxes, calls-to-action, an dadding new identities to existing information. Javascript increases the interactivity of a site.

02. What are Single Page Apps (SPAs)? How do they differ from traditional multi-page websites?

A single page app (SPA) is a web application that interacts with users dynamically by drawing new data from the web server instead of loading entire web pages into the browser. The hope of this kind of application is to make it as fast as possible and to make transitions feel seamless.

SPAs load all needed HTML, JS and CSS in a single page load, or add resources as needed in response to user actions.

Other ways they are different from regular websites include:

    They present a challenge in terms of search engine optimization because not all content is loaded to the page.
    Because they are by definition a single page, users are not able to effectively use their forward or back buttons to navigate.
    Traditional analytics tools have trouble tracking the user’s activity on these pages.

Web designers focus on the appearance or design of a website or application. They have studied graphic design, color theory, and how information should flow for users. Some aspects of web design touch on user experience, such as information flow or hierarchy. Some of the tools that we designers use include Adobe Illustrator, Photoshop, or other tools to wire frame sites. Web designers should also know HTML, CSS, and JS.

Web developers, on the other hand, focus on the function of a website or application. Front end developers work on functionality for the part of a site or app that you see. They use HTML, CSS, and JS skills to develop and work closely with web designers. Back end developers work on the system or database behind a site, and use programming languages or frameworks like Ruby on Rails or Python.

04. Distinguish the difference between Site Relative, Document Relative, and Absolute Paths.

Absolute paths always include the http://www., whereas relative links point to a specific file (document relative) or a file path. A relative link takes a user to a location on the current site. Relative links are used to link to files (document relative) or pages (relative) within your site. Absolute paths lead to a location on another website.

Relative paths include:

    Index.html

    /graphics/image.png

    /help/articles/how-do-i-set-up-a-webpage.html

Absolute paths include:

    http://www.mysite.com

    http://www.mysite.com/graphics/image.png

05. What does it mean for a company to be Agile? What is the purpose of using Agile and what are its pros and cons?

An agile company is one that can quickly adapt to market changes. The company has to be fast-moving and flexible, be able to respond quickly to unexpected challenges, be a change leader, listen to customer feedback, have a competitive advantage, and utilize a flat organizational structure for rapid communication.

The purpose of using agile is to be able to reinvent quickly for continued success. An agile business can make decisions quickly and implement change as a result.

The pros of agile are flexibility, quick feedback and fewer defective products. The cons of Agile include lack of documentation, scope creep, extended time on projects and lack of predictability.

07. What is the difference between jpg, gif, png and SVG images?

    jpg: JPG or JPEG stands for Joint Photographic Experts Group. A jpg is optimized for photographs. They work with compressing color so are great for photos and complex digital images. JPGs are “lossy raster” files, so they lose a small amount of information everytime the file is saved. A high quality jpg is suitable for print.
    gif: GIF stands for Graphics Interchange Format. This is a bitmap format and supports up to 8 bits per pixel which limits the color palette to 256. This means gifs are not good for photos and these days are mostly used for simple animated memes.
    png: PNG stands for Portable Network Graphic. PNGs were designed for use on the web and are the most commonly used image compression format on the internet. They are not for printing professional images. PNGs are raster graphics that work well with line art images fnd photos for websites but can result in large file sizes. PNGs also offer alpha transparency which means each individual pixel has its own level of opacity. There are two types of PNGs: PNG-8 (8 bits per pixel) and PNG-24 (24 bits per pixel).
    SVG: SVG stands for Scalable Vector Graphics. SVGs are vector graphics, meaning they can scale truly to any size. They support animation and their behaviors are defined in XML files so they can be searched, indexed, scripted, and compressed. Vector graphics are good for logos and line art because they can be infinitely resized without losing quality or increasing in size. SVGs are best when creating icons and line art for high resolution displays.

Testing environments are servers that allow you to run the test cases you have defined. The test environment includes more than just setting up a server; it includes hardware and network configuration.

Web accessibility is important because by making a website accessible, you are ensuring that all potential users, including people with disabilities, have a good user experience and are able to easily access information. Implementing accessibility best practices also improves the usability of the site for all users. Finally, it ensures compliance with web and governmental guidelines.

13. What is a CSS Preprocessor? What are some examples? In React, what are styled components? How do Styled Components violate separation of concerns?

A CSS Preprocessor is a program that allows you to generate CSS from the preprocessor's own unique syntax. Most CSS preprocessors will add some features that don't exist in pure CSS, such as mixin, nesting selector, inheritance selector, etc. Some examples are Sass, LESS, and Stylus.

In React, styled-components utilizes tagged templates to style your components. It removes the mapping between components and styles, meaning that when defining your styles, you're actually creating a normal React component with styles attached.

Separation of concerns is about each compoentnt being its own concern and containing all the logic, styles, etc. needed. Styled-components adds the style to the individual component thus violating the separation of concerns.
